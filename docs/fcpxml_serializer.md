# Module 9: Minimal FCPXML Serializer / File Export

## Goal

Module 9 serializes a Module 8 `abstract_fcpxml_design` into a minimal `.fcpxml` file.

This is the first stage that writes an FCPXML file, but it still does not read video, inspect media, transcode, render, export finished video, or launch Final Cut Pro.

## Input Contract

The serializer only accepts design objects where:

- `status == "designed"`
- `target_profile.target_id == "fcpxml"`
- `xml_generated == false`
- `project_file_written == false`
- `media_files_read == false`
- `resources.sequence_format` is present
- `resources.assets` is present
- `sequence_design.spine` contains at least one clip
- every clip `ref` points to a declared asset

Blocked or partially validated designs must not be serialized.

## Output

The serializer writes a `.fcpxml` file containing:

- XML declaration
- `DOCTYPE fcpxml`
- root `<fcpxml version="1.10">`
- `<resources>` with one sequence format and asset entries
- `<library><event><project><sequence><spine>`
- minimal `<asset-clip>` elements
- narration text as minimal marker metadata under the first `asset-clip`

The write result reports:

- `fcpxml_file_written`
- `media_files_read`
- `editor_launched`
- `video_export_performed`
- `bytes_written`

## Time Policy

Module 9 consumes rational seconds generated by Module 8. It does not recompute frame alignment and does not round times.

Sequence duration is calculated with `Fraction` from clip `offset + duration`.

## Marker Policy

Narration markers must include a real `timeline_start`.

`source_timeline_item_id` is traceability only. It must not be used to infer a marker time.

During serialization:

- The serializer finds the single clip whose timeline range covers `timeline_start`.
- The marker is attached to that clip.
- The marker `start` attribute is converted to clip-relative time: `marker.timeline_start - clip.offset`.
- Marker duration is one sequence frame.

The serializer blocks:

- markers without `timeline_start`
- markers in gaps between clips
- markers outside sequence duration
- markers that match more than one overlapping clip

It never places all markers on the first clip and never defaults marker start to `0s` unless the marker truly starts at the matched clip's beginning.

Module 8 must preserve an empty marker `timeline_start` when the narration cue does not provide one. Module 9 then blocks serialization with `missing_marker_timeline_start`.

## Boundary

Module 9 does not:

- Read, probe, transcode, cut, render, or export video.
- Launch or control Final Cut Pro.
- Verify that Final Cut Pro can import the file.
- Relink media or check whether `source_file` exists.
- Support non-millisecond frame rates that Module 8 blocks.
