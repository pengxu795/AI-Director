# AI 短剧编导助手（MVP）PRD

## 1. 项目目标

开发一个 AI 短剧编导助手，将短剧二创从 2-3 小时缩短到 5-10 分钟。

产品定位：

- AI 负责理解剧情、写解说、匹配镜头。
- 剪映负责字幕、配音、时间线、导出。
- 本项目先产出统一 JSON 中间格式，后续再对接剪映工程或自动导入方案。

## 2. 开发原则

1. 不一次性开发全部功能。
2. 采用模块化开发。
3. 每个模块完成后都能独立测试。
4. 优先完成 MVP。
5. 每完成一个模块，确认后再进入下一模块。
6. 所有模块之间低耦合。
7. 所有中间数据统一采用 JSON。

## 3. MVP 工作流程

```text
导入短剧
↓
读取字幕
↓
AI 分析剧情
↓
AI 自动生成解说稿
↓
用户可修改解说稿
↓
AI 将解说稿逐句匹配到原字幕
↓
生成镜头时间线
↓
导出时间线
```

## 4. 模块规划

### Module 1：字幕解析

目标：读取 SRT 字幕并输出 JSON。

输入：

- SRT 文件

输出：

```json
[
  {
    "start": "00:02:15.200",
    "end": "00:02:18.800",
    "text": "你不是我的亲生女儿"
  }
]
```

要求：

- 保留时间码
- 保留文本
- 保持原顺序
- 可独立测试

### Module 2：剧情分析

输入：

- 字幕 JSON

输出 JSON：

- 人物关系
- 主线剧情
- 冲突
- 爽点
- 反转
- 高潮
- 不可剧透内容

### Module 3：解说稿生成

输入：

- 剧情分析 JSON

输出：

- 约 2 分钟解说稿
- 650-750 字
- 抖音短剧解说风格
- 开头 3 秒有钩子
- 每 8-12 秒一个反转
- 结尾留悬念

支持风格：

- 抖音版
- 视频号版
- 爽文版

### Module 4：镜头匹配

输入：

- 解说稿
- 字幕 JSON

目标：

- 不能只做关键词搜索
- 需要理解剧情语义
- 输出逐句解说对应的原片字幕时间段

输出示例：

```json
[
  {
    "sentence": "女主终于发现孩子不是自己的",
    "episode": 4,
    "start": "12:31.200",
    "end": "12:39.800"
  }
]
```

### Module 5：时间线生成

输入：

- 镜头匹配结果

输出：

- `timeline.json`

要求：

- 采用统一时间线格式
- 后续可转换为剪映工程、Premiere、DaVinci、XML、EDL

### Module 6：导出

第一阶段导出：

- `timeline.json`
- 解说稿
- 镜头清单

第二阶段研究：

- 生成剪映可导入工程
- 或通过自动化导入剪映

## 5. 第一版 UI
